# EDI Workflow Demo — Copilot Persistent Context

## Project Purpose

A portfolio/resume demonstration project that simulates a real-world healthcare EDI
(Electronic Data Interchange) workflow system. It ingests X12 837 (claim) and 835
(payment/ERA) files, parses them, persists structured data to PostgreSQL, runs
validation rules, and exposes a REST API consumed by a Vue frontend.

## Architectural Patterns & Conventions

### Package Structure

- `controllers` — REST endpoints only, no business logic
- `services` — business logic, file I/O, orchestration
- `parsers` — stateless EDI parsing (one class per transaction set)
- `repositories` — Spring Data JPA interfaces only
- `models` — JPA entities with Lombok
- `config` — `@ConfigurationProperties` classes

### Naming Conventions

- Controllers: `XxxController.java` → `@RequestMapping("/api/xxx")`
- Services: `XxxService.java` → `@Service`
- Parsers: `EdiXxxParser.java` → `@Component`
- Repositories: `XxxRepository.java` → `extends JpaRepository<X, Long>`

### Key Decisions Made

1. **`@EnableConfigurationProperties(EdiIntakeProperties.class)`** in
   `BackendApplication.java` — not `@ConfigurationPropertiesScan`
2. **`ddl-auto: validate`** — schema is managed manually via SQL migrations,
   not auto-generated by Hibernate
3. **Lombok `@Data`** generates all getters/setters — do NOT write them manually
4. **`@PrePersist`** in `Claim.java` sets `createdAt`, `updatedAt`, and defaults
   for `claimStatus` ("RECEIVED") and `ediType` ("837")
5. **De-duplication** in ingest: `claimRepository.existsByClaimId()` skips
   duplicate claims before saving
6. **Required field guard** in ingest: claims missing any of
   `claimId`, `patientId`, `providerId`, `payerId`, `serviceDate`,
   `procedureCode`, `diagnosisCode`, or `billedAmount` are skipped (not errored)
7. **835 detected but skipped** — `EdiTransactionClassifier` reads `ST*xxx`
   to identify transaction set; 835 returns "Skipped" until Stage 5

### EDI Parsing Notes

- Segment delimiter: `~`
- Element delimiter: `*`
- Component delimiter: `:`
- 837 parser handles `NM1*IL` (subscriber) and `NM1*QC` (patient) both as
  patient ID — loop-level NM1s captured before CLM segment are applied to
  the next CLM
- 837 parser reads: `CLM`, `NM1` (IL/QC/82/PR), `DTP*472`, `SV1`, `HI`
- 835 parser: **NOT YET BUILT** — sample file is available for reference

## Known Issues / Lessons Learned

- Every new `.java` file **must** have its `package` declaration match its
  directory path exactly — Maven will throw "file does not contain class" otherwise
- Lombok must be in `pom.xml` as `<optional>true</optional>` — without it,
  `@Data` silently fails and getters are missing at compile time
- Spring Boot `spring-boot:run` exit code 1 with no detail = check for
  missing `@EnableConfigurationProperties` or a bean that failed to construct
- `SampleEdiIngestService` depends on `Edi837Parser`, `EdiTransactionClassifier`,
  and `ClaimRepository` — all must exist and compile before the service compiles
